### 2. Provider tiering

**(a) Premium intelligence (smart-money / labels)**
- **Nansen** — use when you need curated top-wallet flows + proprietary entity labels with an official MCP; the alpha layer for an indie build (Pro $49/mo).
- **Arkham** — use when you must de-anonymize a wallet to a *named* real-world entity or trace CEX/fund flows; accept the access-application + no-official-MCP friction.

**(b) Query / warehouse platforms**
- **Dune (Query API)** — use for arbitrary custom SQL over decoded multi-chain data and reusing community queries; analytics, not realtime.
- **The Graph** — use when you need a bespoke indexed schema (custom subgraph) or cheap standardized Token API.
- **Allium** — use only at enterprise scale when you want decoded tables in your own warehouse + low-latency realtime; not indie-budget.

**(c) Raw multi-chain APIs**
- **Covalent/GoldRush** — use for normalized genesis-to-now archival across 100+ chains incl. Solana/Bitcoin (mind the archived MCP).
- **Moralis** — use for mature EVM+Solana balances/tx/NFT/PnL endpoints (mind the Cortex retirement + branding requirement).
- **Bitquery** — use for realtime DEX trade/OHLCV streaming + new-pair/mempool signals.
- **Dune Sim** — avoid for new builds (sunsetting Aug 1 2026).

**(d) Free DeFi / market data**
- **CoinGecko** — use as the default keyless price/market/OHLC backbone via official MCP.
- **DexScreener / GeckoTerminal** — use for keyless new-pair discovery and pool OHLCV/trending.
- **DeFiLlama** — use for free TVL/yield/stablecoin/fees signals.

**(e) Analyst-LLM wrappers**
- **Surf (AskSurf.ai)** — use when you want one credit-metered key to fan across CEX+DEX+on-chain SQL+social+prediction-markets+AI-classified news, accepting aggregator/durability risk.
- **Glassnode** (borderline) — use for the analyst-grade macro indicator layer (SOPR/MVRV/NUPL) with official MCP, knowing real API access is Professional-tier-gated.

### 4. Key insights for architecture

- **Provider-adapter normalization is mandatory.** Every provider uses a different auth header (`apikey`, `X-Dune-API-Key`, `X-Sim-Api-Key`, `NANSEN-API-KEY`, Bearer, JWT, x402) and shape (REST/GraphQL/SQL/WebSocket). Build a thin normalization layer keyed to one internal schema (token, wallet, flow, OHLCV) so providers are swappable — critical given Sim's shutdown and GoldRush/Moralis MCP churn.
- **MCP-first for data reads, SDK-first for execution.** The data layer is overwhelmingly MCP-ready (CoinGecko, Dune, Nansen, Glassnode, Bitquery, Allium, The Graph all ship official MCP). The *act* layer is SDK/adapter-shaped (GOAT, AgentKit, Hummingbot+its MCP) and holds keys/signs tx — keep it isolated behind explicit approval gates.
- **Layer free-vs-paid deliberately.** Default free keyless tier (CoinGecko MCP, DexScreener, GeckoTerminal, DeFiLlama free, The Graph 100k/mo) for breadth; reserve paid credits for the alpha layer (Nansen Pro $49/mo, Glassnode, Bitquery Commercial) only on cache-miss / high-value queries.
- **A caching + credit-budget layer is non-optional.** Almost every paid provider is credit/CU-metered with premium endpoints that burn fast (Nansen Agent Expert 750cr/call, Profiler 500cr; Dune by datapoint; Glassnode credits). Cache by (provider, endpoint, args, TTL) and track per-call `credits_used` to enforce a budget ceiling.
- **Signal/alert engine is yours to build, but patterns are abundant.** No OSS repo is a turnkey end-to-end analytics agent. Steal swap-decoding from handi-cat, RPC wallet-delta alerting from Insider-Monitor, and narrow-signal MCPs (whale-tracker, fear/greed, new-pool) from kukapay's catalog; wire them into your own rules engine.
- **Smart-money/labels are the hardest-to-replicate moat — and proprietary.** Nansen and Arkham labels can't be reproduced cheaply; treat them as a paid premium signal, not something to rebuild. Surf bundles labels too but inherits upstream accuracy/latency.
- **Realtime vs analytics is a hard split.** Dune Query, Glassnode, DeFiLlama are batch/lagged; Bitquery streams, DexScreener/GeckoTerminal, and (formerly) Sim are realtime. Route latency-sensitive signals (new-pair, mempool) to streaming providers and backtest/regime signals to the SQL/warehouse side.
- **Execution is largely solved by OSS; pick by venue.** Hummingbot (+ official MCP) for DEX/CEX spot, NautilusTrader for deterministic backtest→live derivs, Freqtrade for CEX legs. Don't build an execution engine — adopt one and feed it your signals.
- **Watch license traps.** Heurist framework is BSL 1.1 (not OSI); Freqtrade/OctoBot are GPL-3.0 copyleft; the three best wallet-tracker repos (Insider-Monitor, handi-cat, whale-mirror) have **no license** — pattern-only, legally unsafe to vendor in.
- **MCP tool-count/endpoint claims drift — verify at integration time.** Verification already corrected Dune (26 not 29 tools) and Nansen (exactly 24); marketing endpoint counts (Surf 83/86/90+, Allium 50/80/150 chains, Bitquery 8 vs 40+) are inconsistent. Probe the live tool list, don't hardcode counts.
